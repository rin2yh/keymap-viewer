# 表示テキスト規約

人間が目にするテキスト 例 PR の summary / test plan、コミットメッセージ、CI のジョブ名・ステップ名、コード内コメント、ドキュメント の散文部分での書き方ルール。

- **散文での丸かっこ禁止**
  「ジョブ名 small」のように丸かっこ無しで書く。補足は読点・コロン・ダッシュなどで表現する。
- **構文要件のあるかっこは対象外**
  以下は丸かっこを使ってよい:
    - markdown link `[text](url)`
    - Go の関数呼び出し `via.Open()`
    - bash の command substitution `$(cmd)`
    - 数式やコード片で構文上必要なもの
